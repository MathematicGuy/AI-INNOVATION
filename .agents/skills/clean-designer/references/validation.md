## Validation Checklist

### Task validation
- Is the primary action obvious?
- Can the user complete the task with fewer decisions or steps?
- Are consequences clear?
- Are errors preventable and recoverable?

### Visual validation
- Is hierarchy visible within five seconds?
- Are related elements grouped?
- Is alignment intentional?
- Is emphasis limited?
- Does the screen remain calm with realistic data?

### Accessibility validation
See `accessibility.md`.

### Feature-preservation validation
- Does every required feature remain available?
- Are all states represented?
- Did subtraction remove meaning, only clutter, or both?
- Did the design create a new hidden dependency?

## Clean Design Scoring Rubric

Score each dimension 0–3: **0 Fails** (major usability/accessibility
problem), **1 Weak** (understandable with effort, substantial friction
remains), **2 Good** (clear and usable, minor improvements remain), **3
Strong** (clear, efficient, accessible, consistent, resilient).

Dimensions: task clarity, hierarchy, information architecture, scannability,
interaction efficiency, feedback, consistency, accessibility, responsive
resilience, information density, feature preservation, strategic
subtraction.

Do not use the total score alone — explain any dimension scoring 0 or 1.
