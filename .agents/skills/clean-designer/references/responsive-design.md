# Responsive Design

Back to [`INDEX.md`](../INDEX.md).

Design for content behavior, not only fixed breakpoints. For every proposed
layout, specify: wide desktop behavior, narrow desktop or tablet behavior,
mobile behavior, wrapping behavior, overflow behavior, and long-content
behavior.

## Preferred Responsive Strategies (in order)

1. Reflow
2. Wrap
3. Stack
4. Collapse secondary detail
5. Enable controlled horizontal scrolling for true comparison tables
6. Truncate only when the full value remains discoverable

Never solve responsiveness by making text unreadably small.
